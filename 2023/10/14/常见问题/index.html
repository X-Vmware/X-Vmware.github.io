量子验证常见问题

QP模块生成：

1、首先在服务端的通讯配置生成qplincense.dat文件
2、生成完成之后，打开服务端目录下QPModule.exe生成易模块即可！

QP加密端不支持系列：

1、不支持有壳软件
2、不支持加密 Python、C#/.Net程序

QP加密端成品对接后运行无登录界面：

1、不支持有壳软件
2、不支持加密 Python、C#/.Net程序
3、可尝试"右键以兼容模式运行，选择win7"
4、如果你是32位程序，可尝试"换个模板，不要使用[QPKeyLoginTemplateX86.dll]模板"
5、可尝试添加数据执行保护，具体过程非常简单百度自行查找怎么添加，添加完成之后需要重启电脑即可

QP加密端无法加密或提示SDK编译失败：

1、请检查你的程序VMP保护标志是否错乱，如果有错乱请删除所有VMP保护标志，不光你的程序有标志可能你的模块也有VMP保护标志都要进行检查,下面是VMP保护标志的格式
2、置入代码 ({ 235, 16, 86, 77, 80, 114, 111, 116, 101, 99, 116, 32, 98, 101, 103, 105, 110, 0 }) ' VMP保护开始标志，找到所有的VMP保护标志并且删除它
3、置入代码 ({ 235, 14, 86, 77, 80, 114, 111, 116, 101, 99, 116, 32, 101, 110, 100, 0 }) ' VMP保护结束标志，找到所有的VMP保护标志并且删除它
4、如果还是加密不上，请检查你的易语言模块有没有这个VMP标志，需要一同去掉即可，如果没有模块源码且你不会反编译请联系小曾解决标价一次200元

QP加密端成品对接后有登录界面,但是登录成功之后闪退：

1、不支持有壳软件
2、不支持对接 Python、C#/.Net程序
3、可尝试"右键以兼容模式运行，选择win7"
4、如果是vs编译，请切换到release重新编译你的程序，后再次加密即可解决
5、如果你的程序一定需要debug版编译，则设置4个选项
    a) C/C++ -> 常规 -> 支持仅我的代码调试:否
    b) C/C++ -> 常规 -> 调试信息格式:程序数据库 (/Zi)
    c) C/C++ -> 代码生成 -> 基本运行时检查:未初始化的变量 (/RTCu)
    d) C/C++ -> 语言 -> 符合模式:否 (/permissive)
    e) 链接器 -> 常规 -> 启动增量链接:否 (/INCREMENTAL:NO)
6、可尝试添加数据执行保护，具体过程非常简单百度自行查找怎么添加，添加完成之后需要重启电脑即可

QP加密端x64 DLL成品对接后有win11 登录界面,但是登录成功之后闪退：

1、升级到升级到win11专业版的操作系统，如果当前是win11专业版不需要升级
1、切换到Release进行编译，确保release版正常运行然后再次成品对接加密
2、控制面板-->系统-->系统信息-->高级系统设置-->高级-->性能-->设置-->数据执行保护-->选择仅为基本Windows程序和服务启动DEP-->确定
3、，然后Win+R-->输入secpol.msc-->选择本地策略-->安全选项-->然后在右侧窗口找到"用户账户控制: 以管理员批准模式运行所有管理员"-->双击--> 选择已禁用-->应用-->确定-->重启电脑-->完成
4、最后重启电脑

出现”程序未授权”解决方案：

1、如果QP加密端加密出的DLL关闭的时候出现"程序未授权"，那么此现象为正常现象，DLL本身的性质造成，但是不影响正常使用，SP也这样！
2、如果你的是x64 DLL，请打开x64兼容选项即可
3、如果是登录之后出现"程序未授权"，这一般由网络波动或本地时间被改动导致出现的不正确登录出现的反破解机制，这个是正常现象，在群文件\验证工具下载[程序未授权修复.exe]修复即可,AB版通用！
    -注意：需要在出现"程序未授权"的电脑修复，不是你的服务器！
    -注意：需要在出现"程序未授权"的电脑修复，不是你的服务器！！
    -注意：需要在出现"程序未授权"的电脑修复，不是你的服务器！！！

使用C++编译自定义模板闪退问题：

编译模板的时候，模板请选择MTD编译，如果选择MT运行库编译，则部分系统可能出现不兼容，打不开，闪退等情况

QP加密端[内置模板]和[外置模板]区别：

1、内置模板可直接进行成品对接，不需要自己重新编译，QP加密端自动替换IP、端口、软件位等，可以自己做好内置模板可直接发给别人使用
2、外置模板需要自己导入验证模块，然后自己添加 验证_初始化的信息，然后编译即可加密(外置模板相对内置模板只是为了更好的调试)
3、外置模板不能发给别人使用，否则别人加密出来的程序会连接上你的服务器而不是他的服务器

QP服务端登录进去不显示在线列表或打开卡死解决方案：

1、一般由于你的不正常或者是多次重复打开导致文件被占用导致
2、启动任务管理员关闭所有的残留QP服务端的进程
3、删除QP服务端的rocksd_base_data文件夹重新打开即可解决

QP服务端关闭后卡进程：

请检查服务端Plugin\Bin文件夹下面的云插件是否是导致QP服务端卡进程原因！

QP服务端假死或是出现异常：

请检查你写的云插件是否发生异常，如果你的云插件发生异常，则QP服务端也会发生异常！

C/C++ x86/x64源码以及成品对接：

1、群文件有C/C++对接的视频教程，从第5个教程开始观看后自行对接！
2、群文件上有云计算/插件/成品对接教程
3、也可以找我进行代对接/成品/服务器配置/C++等，一次收费200元！

QP加密出来的软件出现”没有在指定的Windows运行”或出现”Cannot load file CurrentBox.dll”等：

1、请先检查你的软件是32位还是64位，如果是32位既可以在64位系统也可以在32位系统运行，如果是64只能在64位系统运行不能在32位运行！
2、加密 32 位程序选择 86 模板，加密 64 位程序选择 64 位模板

QP加密端加密出来的软件出现”Cannot load file CurrentBox.dll”：

1、由于系统可能被病毒感染或者是缺少什么组件造成的原因
2、解决方案：把QP加密端/Tmp目录下的CurrentBox.dll与你的程序放到一起即可

QP加密端qplincense.dat文件不存在：

1、请在服务端->通讯配置->生成qplincense.dat
2、将生成完成的qplincense.dat拷贝到加密端目录下即可

QP加密端出现红色提示 “错误(1) 错误(2) 错误(3)”等：

1、qplincense.dat文件被篡改，可能是由于病毒或杀毒篡改，或者是自己操作误改！
2、请重新将服务端的qplincense.dat文件拷贝到加密端目录覆盖或重新生成qplincense.dat文件即可解决！

卡密到期时间软件没有结束：

1、先生成个分钟卡，然后登录你的软件，然后观察1分钟之后会不会结束，如果正常结束则说明没问题！
2、如果你的软件是注入型的，你注入的DLL是没有加验证的到期后是不会掉的，但是你的主程序加了验证，到期后一定会掉，解决方案对注入DLL也加个验证

卡密没有到期软件就结束：

如果开启心跳了，请确保网络通畅，并且电脑没有休眠，一般情况下是不会结束的

PIC盾数据出错：

1、请用一下原版例子会不会出错，如果原版例子不会出错那可能你代码可能没有写好
2、如果还会出错，请重新解压一下给你发的定制版和定制版服务器，.dat文件重新生成，再次尝试
3、如果还会出错请联系作者

服务端显示”心跳超时自动下线”：

1、"心跳超时自动下线" 此现象一般为正常现象
2、一般是你的软件在退出的时候没有调用退出登录导致，如果调用退出登录就不会出现"心跳超时自动下线"现象
3、如果是成品对接出现"心跳超时自动下线"现象，则为正常现象
3、如果不出现"心跳超时自动下线"就是不正常了，出现"心跳超时自动下线"现象才是正常的！

解绑提示”需要过XXX时间后再进行解绑”：

1、这种情况一般由于程序异常退出或者是强制关闭而没有调用验证退出登录函数导致，一般情况下等待6分钟即可
2、如果是源码对接请检查软件在退出的时候是否有调用 验证_退出登录 函数，如果没有调用，请在你的软件退出时加上此函数
3、如果你想减少等待时间，请把服务端的 心跳未响应跳超时成30，心跳间隔调成2000，这样如果异常关闭程序只需要等待30秒即可解绑
4、你也可以在你服务端后台自己手动解绑或在线管理强制下线
5、如果是成品对接验证，而你又不想出现这种情况，请按照步骤3进行操作
6、如果你需要在不同的机器码的电脑使用同一张卡密请打开一卡多用即可

QP加密端加密之后软件运行缓慢并且占用CPU大解决方案：

1、保护选项全部关闭，只保留检增强脱壳检测即可
2、如果是成品对接，找到自定义模板的心跳线程，并且把心跳线程那个 延时 改成精易模块的 程序_延时 ，即可不占CPU
3、尝试以上2点如果不行，更换到引擎1再次尝试

一键成品对接出现更新地址链接打不开情况解决方案：

1、由于成品对接会阻塞线程，所以造成打开网址无响应
2、解决方案：请把 自定义模板的 打开指定网址函数 更改成 运行("explorer https://www.baidu.com"，假，#普通激活) 即可

QP加密端出现SDK失败、成品对接弹窗2次，找不到问题等解决方案：

1、退出所有杀毒
2、关闭你的Windows 自带的windows defender杀毒软件，或者是把你的C盘设置排除选项
3、请检查你的软件有没有自启动代码，如果有自启动代码删除即可(重点)
4、你的软件可能会检查当前有没有管理员权限，如果没有管理器权限会关闭当前进程然后再重启一份有管理员权限的进程，所以会弹出2个登录框
    1、解决方案1：升级到专业版的操作系统，然后Win+R --> 输入secpol.msc -->选择本地策略 -->安全选项 -->然后在右侧窗口找到"用户账户控制: 以管理员批准模式运行所有管理员" -->双击  --> 选择已禁用  --> 应用  --> 确定  --> 重启电脑  --> 完成
    2、解决方案2：请以管理员身份运行即可

服务端换版本之后制卡无效果：

1、更新服务端的时候请不要直接替换数据库，否则出现制卡无效果
2、先检查当前服务端版本和需要更新的服务端版本的数据库字段一不一致，如果一致则需要直接替换覆盖数据库即可
3、如果不一致，请先将旧版本的卡密 和 用户导出来，然后新版服务端导入进去，另外软件位需要自己手动新建，需要和原来的软件位保持一致即可、另外卡规则、代理也需要自己手动新建也要与旧版本保持一致即可

客户端出现”无效授权信息”:

1、无效授权信息是因为你没有把授权文件(xxxx.qvmp)放到服务端下的VMP授权文件
2、如果放进去了，还出现这种情况，请重启服务端

用户注册或充值取不到附加数据：

1、用户本身那个附加数据需要你手动加上去的，注册卡密和充值卡密并不会把卡密的附加数据覆盖到用户的附加数据
2、如果你要实现那种用户注册或充值，自动覆盖用户附加数据，请参考客户端文件夹下面的那个"会员制.e"
3、另外提醒：用户的绑定数据跟用户附加数据不一样的，绑定数据是用户注册的时候指定绑定的数据，比如用户注册的时候绑定一个游戏号等
4、在客户端用户充值 卡密，则"会员制.e"会自动取到最后一张充值卡的附加数据，也可以用于升级

加密端出现缺少qplincense.dat：

1、首先在服务端的通讯配置生成qplincense.dat文件
2、生成完成之后，将qplincense.dat拷贝到加密端目录即可

低版本 的云插件放到 高版本服务端(v32及以上的版本)：

原因：新版本的服务端插件标识会变化，所以给出了以下解决方案：
1、将DLL换自定义云插件标识，在源码里面换自定义云插件标识，然后编译出来
1、编译出来之后，将云插件文件名改成 xxx.qp.dll 的后缀名
2、将dll拷贝到新版本即可

量子成品加密出来打开卡死问题：

1、退出电脑的"有道词典","百度翻译"等占用快捷键的程序(原因可能是有道词典HOOK了一些全局消息函数，导致DLL界面消息阻塞)
2、如果不行，把后台全部进程都退出并且切换到纯英文输入法再次尝试
3、如果再次不行请换台电脑或虚拟机进行尝试，依次排查原因

量子通用更新步骤：

0、如下方法更新不会导致你的客户掉线！！！
1、复制服务端目录下的 【qplincense.dat】，【ser_config.ini】，【rocksd_base_data】，【VMP授权文件】 到新版服务端目录下覆盖即可
2、打开群文件"量子数据库一键更新.exe" --》选择旧版和新版数据库路径(数据库名：data.mdb) --》点击检测并增加字段 --》提示增加了字段(如果没反应说明数据库已更新过) --》再把 旧版数据库 拷贝到 新版服务端 目录下覆盖即可
3、云插件更新(没有云插件忽略即可)：如果你有写好的云插件，请把云插件一并拷贝过去(注意云插件标识变动问题)，如果云插件标识变动了请用自定义云插件标识
4、双击打开QPServerRelease.exe即可
